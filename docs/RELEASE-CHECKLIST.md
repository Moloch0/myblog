# RELEASE CHECKLIST

用于每次发布前后检查，避免回归和发布事故。

## A. 发布前（必做）

- [ ] 已阅读核心规范：`_posts/2026-03-09-config-zh-cn.md`
- [ ] 本次变更范围明确，且有任务记录（`docs/TASK-TEMPLATE.md`）
- [ ] 未误改关键配置：`lang`、`baseurl`、评论与统计配置
- [ ] 未引入构建产物：`_site/`、`.jekyll-cache/`
- [ ] 未引入缓存文件：`__pycache__/`、`*.pyc`
- [ ] 新增/修改文章已直接在 `_posts/` 维护
- [ ] 新增文章 Front Matter 完整（至少 `title`、`date`）
- [ ] `git diff` 已人工审阅，确认无意外删除

## B. 构建与测试

- [ ] 本地启动通过：`bash tools/run.sh`
- [ ] 本地测试通过：`bash tools/test.sh`
- [ ] 如跳过测试，已在任务记录中注明原因与风险
- [ ] 若有脚本改动，命令示例可运行

## C. 发布执行

- [ ] 提交信息清晰（包含变更范围与目的）
- [ ] 已推送到目标分支
- [ ] CI/Pages 工作流状态成功
- [ ] 站点首页可访问
- [ ] 至少抽查 1 篇旧文 + 1 篇新文页面

## D. 发布后回归（建议 10 分钟内）

- [ ] 导航/分类/标签页正常
- [ ] 评论区加载正常（giscus）
- [ ] 统计代码加载正常（goatcounter）
- [ ] 图片放大、播放器等自定义脚本无报错
- [ ] 移动端和桌面端布局均正常

## E. 回滚触发条件

满足任一条件应触发回滚或热修：

- [ ] 首页或文章页 5xx/空白
- [ ] 链接结构异常导致历史文章不可达
- [ ] 评论或统计脚本导致主要页面报错
- [ ] CI 通过但线上出现严重功能缺失

## F. 发布记录（每次填写）

- 发布日期：
- 发布人：
- 提交哈希：
- 变更摘要：
- 线上验证结果：通过/失败
- 回滚：无/已回滚（原因）
